# Card Grading

> Submit a card for grading (fee + wait). Its grade comes from **centering** (an
> RNG fixed when the card was pulled) and **condition** (preserved by sleeve time,
> degraded by time spent exposed). Grading consumes one copy and returns a
> **slab** with a PSA-style 1–10 grade, sellable at a grade-multiplied value.

## Motivation
The payoff for sleeves: protect a card → grade it → high grade → big value. Adds
a skill/luck loop (pull good centering, preserve it, time your submission).

## The grade model (the core — please confirm)
Two inputs per card, both in [0,1]:
- **Centering `c`** — rolled once when the card is first pulled (uniform RNG),
  then fixed forever. This is the card's intrinsic ceiling; you can't change it.
- **Condition `cond`** — starts at **1.0** (mint) and **decays only while the card
  is *exposed* (not sleeved)**. Sleeving pauses the decay; unsleeving resumes it.
  `cond = clamp(1 − exposedMs / EXPOSURE_CAP, 0, 1)`, `EXPOSURE_CAP = 60 min`.

**Grade** (deterministic given c + condition at submission) — **Gate 1: condition
weighted higher than centering**:
`grade = clamp(round(1 + (0.35·c + 0.65·cond)·9), 1, 10)`.

- The only RNG is the centering roll at pull. Everything else is your choices
  (sleeve fast, keep sleeved, submit while fresh).
- Fresh + sleeved (cond≈1): grades **7–10**; centering decides where (≈16% hit a
  10, ≈48% ≥9). Preservation reliably gets you into high-grade territory.
- Left exposed (cond→0): grades collapse to **1–4**. Neglect is punished.
- Condition is **frozen at submission** (the card's "in the grader's hands").

_Confirmed (Gate 1): 0.35 centering / 0.65 condition, `EXPOSURE_CAP = 60 min`,
uniform centering._

## Economy (Gate 1 decisions: steep multipliers, fee + wait queue, slab sells)
- **Fee**: $20 per submission (flat for MVP).
- **Turnaround**: a grading queue with a countdown (~15–25s, mild tier scaling),
  jobs run **in parallel** (unlike the single-file sell queue). Completes via the
  existing 250ms ticker even off-tab.
- **Grade multiplier** on the card's market price (steep/realistic):
  `10→×6, 9→×3, 8→×1.7, 7→×1.3, 6→×1.15, 5→×1, 4→×0.9, 3→×0.8, 2→×0.7, 1→×0.6`.
  Low grades are a **loss** vs raw + you paid the fee → grading is a real gamble.
- **Slab value** = `card.price × multiplier(grade)`; **sell** = slab value × 0.7
  (same haircut), via the existing `pendingSales` queue.

## Behavior
1. **Submit** (card detail modal): "🔬 Grade — $20" if you own ≥1 and can afford.
   Charges $20, **consumes one copy**, freezes condition, creates a grading job
   (centering+condition+grade computed now, revealed on completion). Toast.
2. **Queue**: jobs count down; on completion the card becomes a **slab**
   `{ grade, value }` in `state.graded`; a toast reveals the grade.
3. **Slabs** live in a new **Binder ▸ Graded** subtab: card image, a `PSA {n}`
   badge, graded value, and a **Sell** button (lists at slab value × 0.7).
   Slabs can't be un-graded.

## Data & state
- **Binder entry** gains condition tracking (set in `addCards`/`addPacks`):
  `centering` (0–1), `pulledAt`, `exposedMs`, `exposeStart` (ts while exposed,
  null while sleeved). `toggleSleeve` updates `exposedMs`/`exposeStart`. Old
  entries/saves are lazily initialised on first grade (roll centering, start now).
- `state.grading: []` — jobs `{ id, uid, card, fee, startedAt, readyAt, grade, value }`.
- `state.graded: []` — slabs `{ id, uid, card, grade, value, gradedAt }`.
- New `game/grading.js`: `rollCentering`, `conditionFrom(exposedMs)`,
  `computeGrade(c, cond)`, `GRADE_MULT`, fee/duration constants.
- Persist `grading`, `graded` (+ binder now carries the extra fields). Old saves:
  default `[]`; binder fields lazily filled. **No `CACHE_PREFIX` bump** (set-card
  cache is unaffected; this is save state).

## UI
- `index.html`: a **Graded** subtab in `#binder-subtabs` + a `#binder-graded` pane
  with `#graded-grid`.
- `modal.js`: the Grade button + a small "centering is luck; sleeve to preserve"
  hint (no spoilers — centering/grade not shown pre-submission).
- `binder.js`: render in-progress jobs (countdown) + slabs (grade badge, value,
  Sell) in the Graded pane; wire Sell.
- `shop.js`: add `processGrading(now)` to the existing ticker; slab-sale toasts.
- `styles.css`: slab frame + `PSA {n}` grade badge (colour-scaled by grade).

## Acceptance criteria

- [ ] Submitting charges $20, consumes one copy, queues a job with a countdown.
- [ ] Job completes → slab in Binder ▸ Graded with the right grade + value; grade
      follows the formula (fresh+sleeved high-centering → 9/10; long-exposed → low).
- [ ] Slab value uses the steep multiplier; selling pays slab value × 0.7.
- [ ] State persists across reload; old saves load and can be graded.
- [ ] No new deps; `node --check` clean; Bun harness + headless Brave green.

## Open questions — resolved at Gate 1
1. Weights → **0.35 c / 0.65 cond** (condition dominates). Cap 60 min, uniform c.
2. Fee → **flat $20**.
3. Queue → **parallel** (each job independent).
4. Multipliers → **low grades are a loss** (×0.6–0.9 below grade 5).

## Research & findings
- **economy.js** (read): `SELL_HAIRCUT = 0.7`, `sellValue` (price×haircut),
  `SELL_DURATIONS` per tier, `sellDurationMs`. The slab sell reuses `sellValue`'s
  haircut on the slab's graded value; grading durations mirror `SELL_DURATIONS`
  (a bit longer). Grade-multiplier + fee live here.
- **store.js** (read): the `pendingSales` queue is the proven pattern — a slab
  sale is just a normal sale entry `{ card, value, … }` (card snapshot carries the
  grade for the toast), so **`processSales` needs no change** and slabs sell
  through the same ticker. `addCards`/`addPacks` create binder entries → add the
  condition fields there. `toggleSleeve` already mutates per-uid → update exposure
  there. `listForSale`/`bulkSell` are unaffected (slabs aren't binder cards).
- **shop.js** (read): one 250ms ticker already drives `processSales` regardless of
  tab → add `processGrading(now)` beside it so jobs complete off-tab too.
- **modal.js / binder.js** (read): modal renders action buttons (add Grade);
  binder has Cards/Sealed subtabs + a click-delegated grid (add a Graded subtab,
  same pattern as Sealed; slab Sell button delegated like `.open-sealed`).
- **Centering per-uid simplification**: the binder stacks copies by uid, so
  `centering`/exposure live on the stack (one representative copy), not per
  physical copy. Grading consumes one copy and slabs it at the stack's current
  centering/condition. Flagged as an accepted MVP simplification.

## Gaps & risks
- **Old saves / pre-existing cards** lack the condition fields → lazily initialise
  on first read (`ensureCondition`: roll centering, `pulledAt=now`,
  `exposeStart = sleeved ? null : now`). Covers migration with no version bump.
- **Exposure math** must include the open interval: `exposed = exposedMs +
  (exposeStart ? now − exposeStart : 0)`. Consuming a copy must NOT reset the
  stack clock (other copies keep aging).
- **Math.random** is fine in app code (only Workflow scripts forbid it).
- **No `CACHE_PREFIX` bump** — set-card cache untouched (this is save state).
