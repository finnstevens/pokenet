# Add a 151 booster box

> Give the **151** set a booster box (it had none — it's a special set that
> shipped as bundles/ETBs in reality, but the user wants a box for gameplay).

## Decision
- `box: { packs: 36, price: 400 }` on the `151` entry. Price hand-set — 151 is a
  high-demand set (36 × $12 pack ≈ $432, slight box discount). Tweakable.

## Implementation
- `data/sets.js`: add `box` to the `151` entry. Everything else (Buy Box button,
  Binder ▸ Sealed box tile, rip/open/unbox) works by convention via the existing
  booster-box system. No code change.

## Results & verification
- `node --check` clean.
- Headless Brave: selecting 151 shows a "$400 · 36 packs" Buy Box; buying banks a
  box in Binder ▸ Sealed with the three box actions.
